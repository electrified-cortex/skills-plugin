# publish — spec

A skill that orchestrates a new release of the skills-plugin. Bumps the plugin version, regenerates the distributed `skills/` tree via the build, updates the CHANGELOG, commits the result, tags it, and (when authorized) pushes.

The publish skill does NOT decide WHEN to release. It executes a release the caller has already decided to cut.

## Scope

**In scope.** Replace ad-hoc release ceremonies with a single skill the agent (or operator) can dispatch to cut a new tagged release of `skills-plugin`. The output is a tagged commit on the configured release branch (default `main`) whose tree is the rebuilt skills distribution.

**Out of scope.**

- Deciding WHEN to release. Caller decides.
- Bumping per-skill versions. The plugin is the unit of release; per-skill version markers, if any, are cache-busters set elsewhere.
- Building the source skills repo. This skill consumes the source tree as configured in `build/config.yaml`; it does not modify the source.
- Cross-repo coordination (notifying consumers, opening PRs in dependent repos). Future skill, not this one.
- Generating release notes. The caller supplies `release_notes`. This skill does not summarize git log.

## Definitions

- **release branch** — the git branch the skill operates on. Default `main`. Configurable via `build/config.yaml` `release_branch` key.
- **plugin version** — the SemVer string in `plugin.json` `version` field. The unit of release.
- **build** — invocation of `pwsh build/build.ps1`, which crawls the source skill tree (per `build/config.yaml`) and writes the dist tree under `skills/`.
- **deny list** — patterns from `build/deny-list.ps1` that must NOT appear in the dist `skills/` tree.
- **dry run** — execute every step EXCEPT the final commit/tag/push. Caller inspects the planned outcome.
- **release commit** — the single commit produced by this skill, containing the bumped `plugin.json`, prepended `CHANGELOG.md`, and regenerated `skills/`.

## Inputs

- `version_bump` — required. One of `patch | minor | major`. How `plugin.json`'s `version` is incremented.
- `release_notes` — required. Human-authored summary of what changed since the last tag. Becomes the new CHANGELOG entry's body.
- `source_root` — optional. Override of `build/config.yaml`'s configured source path. Default: read from config.
- `dry_run` — optional boolean. Default false. When true, runs every step except commit/tag/push.

## Outputs

- New commit on the release branch with:
  - `plugin.json` `version` bumped per `version_bump`.
  - `plugin.json` `built` set to today's UTC date (`YYYY-MM-DD`).
  - `skills/` regenerated by `build/build.ps1` against `source_root`.
  - `CHANGELOG.md` prepended with `## [<new-version>] - <today>` + `release_notes`.
- A new annotated git tag `v<new-version>` on the commit.
- Status report: previous version, new version, files changed, build report (skills count, files included, files denied), push result.

## Requirements

R1. **Pre-flight refusal — dirty working tree.** If the working tree has uncommitted changes outside `skills/`, `plugin.json`, `CHANGELOG.md`, the skill MUST stop with status `dirty_working_tree` and list the offending paths.

R2. **Pre-flight refusal — wrong branch.** If the current branch is not the configured release branch, the skill MUST stop with status `wrong_branch` and surface the current branch.

R3. **Pre-flight refusal — bad config.** If `build/config.yaml` is missing or unparseable, the skill MUST stop with status `bad_config`.

R4. **Pre-flight refusal — bad prior version.** If the most recent tag's version does not parse as SemVer, the skill MUST stop with status `bad_prior_version`.

R5. **Version computation.** The skill MUST parse `plugin.json` for `version` and compute the new version per `version_bump` strictly per SemVer rules.

R6. **Build invocation.** The skill MUST invoke `pwsh build/build.ps1`, confirm exit code 0, and capture the build report. If the build's validation pass (per `build/deny-list.ps1`) reports any deny-pattern file in the dist tree, the skill MUST stop with status `build_validation_failed` and surface the offending paths.

R7. **plugin.json update.** Set `version` to the new SemVer. Set `built` to today's UTC date in `YYYY-MM-DD`.

R8. **CHANGELOG update.** Prepend a new section `## [<new-version>] - <YYYY-MM-DD>` followed by `release_notes`. All prior entries preserved verbatim.

R9. **Stage + commit.** Stage exactly: `plugin.json`, `CHANGELOG.md`, `skills/` (full tree). Forbidden: `git add -A`. Commit message: `release: v<new-version>`.

R10. **Tag.** Create an annotated tag `v<new-version>` pointing at the release commit, with message `v<new-version>`.

R11. **Push (conditional).** If `dry_run` is true, the skill MUST stop and return the dry-run report. Otherwise it MUST push the release branch and the tag. If push fails, the skill MUST stop with status `push_failed`, surface the git error, and leave the local commit + tag intact.

R12. **No force-push.** Force-push is forbidden. On push divergence, surface to the operator.

## Constraints

- The skill MUST NOT decide whether to release. It executes a release the caller has decided to cut.
- The skill MUST NOT auto-generate `release_notes`. The caller authors prose.
- The skill MUST NOT modify `skills/` by hand. The build is the source of truth.
- The skill MUST NOT bump per-skill versions inside `plugin.json` (no per-skill version map exists; if internal cache-buster markers exist on individual skills, they are out of this skill's scope).
- The skill MUST NOT amend or rewrite history of prior commits.
- The skill MUST NOT skip pre-flight refusals (R1–R4) on caller request. They are mandatory.

## Dependencies

- `build/build.ps1` exists and exits 0 on success.
- `build/config.yaml` defines a valid `source` block, optionally a `release_branch` key.
- `build/deny-list.ps1` is loaded by `build.ps1` and applied during validation.
- Git working tree is on the configured release branch.
- `pwsh` (PowerShell 7+) on PATH.

## Bailout

- Any pre-flight refusal (R1–R4) stops before any mutation.
- `build_validation_failed` (R6) stops before any commit.
- `push_failed` (R11) after a successful local commit/tag leaves both intact for operator inspection.
