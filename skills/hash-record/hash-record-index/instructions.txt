Params:
`repo_root` (required): abs path to repo root containing `.hash-record/`.

Procedure:
Run every step with named tool. Don't summarize or plan.

0. Validate: `repo_root` contains `..` or shell metacharacters → output `ERROR: invalid repo_root`, stop.

1. Verify store: Bash `test -d "<repo_root>/.hash-record"`. Absent → output `CLEAN`, stop.

2. Collect hash dirs: Bash:
   ```bash
   find "<repo_root>/.hash-record" -mindepth 2 -maxdepth 2 -type d \
     ! -name '.prune' ! -name '.*'
   ```
   Each path: `<repo_root>/.hash-record/<shard>/<full-hash>`. None → output `CLEAN`, stop.

3. Per hash dir:
   a. Bash:
      ```bash
      find "<hash_dir>" -type f -name "*.md" ! -name "manifest.yaml"
      ```
   b. Read each leaf. Extract `file_path` from YAML frontmatter. Absent/invalid frontmatter → skip silently.
   c. Collect non-empty `file_path` values. Deduplicate. Sort lexically.
   d. `manifest.yaml` exists → Read `file_paths`. Matches collected set exactly → skip.
   e. `file_paths` empty → skip; don't create empty manifest.
   f. Union existing `file_paths` (if any) with collected. Deduplicate. Sort.
   g. Write `<hash_dir>/manifest.yaml` (Write tool), exact content:
      ```yaml
      file_paths:
        - <path-1>
        - <path-2>
      ```

4. Count hash dirs with `manifest.yaml` written.

5. Output: count=0 → `CLEAN`; else → `indexed: <count>`

Output (one line):
`CLEAN` — no dirs needed indexing (all up to date or store absent).
`indexed: <count>` — <count> manifest files written.
`ERROR: <reason>` — pre-exec failure (e.g., `repo_root` not found, path-traversal rejected).

Rules:
Never modify leaf `.md` records; read-only.
Never delete — overwrite outdated `manifest.yaml` in-place; don't delete+recreate.
Write ONLY to `manifest.yaml` at hash-dir level.
Sole writer of `manifest.yaml`; no other skill writes manifest.
Merge existing `file_paths` — never lose indexed paths.
Skip leaves with missing/malformed frontmatter silently; never raise on bad records.
Don't follow symlinks in `.hash-record/`.
Reject `repo_root` with `..` or shell metacharacters.
Don't write manifest for hash dirs with no leaf records with valid `file_path`.
