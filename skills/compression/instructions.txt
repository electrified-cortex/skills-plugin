Input: `<file-path>` [`--tier <lite|full|ultra>`] [`--source <src> --target <dst>`] [`--hygiene-verdict <path>`] (default tier: ultra)
Blurb ‚Üí skip gate, step 4

Modes:
Source‚ÜíTarget: `--source X --target Y` ‚Üí read X, compress, write Y. No git check. X untouched.
In-place (default): compress directly. Git check required.
Fallback: if untracked/dirty and no `--target`, create `<file>.compressed` alongside.

Phase 0 ‚Äî Source Hygiene Verdict:
If `--hygiene-verdict <path>` provided: read file. Each line:
- `CLEAN: <file>` ‚Äî no issues
- `FINDINGS: <file> ‚Üí <record-path>` ‚Äî issues found; record at path
- `ERROR: <file> ‚Üí <reason>` ‚Äî check failed
Hold for output. No content action ‚Äî verdict is informational.

Steps:
1. Parse path + tier (default ultra) + mode. Blurb ‚Üí step 4.
2. Gate (skip if `--source`/`--target` provided):
   - cd to file's parent dir, `git status --porcelain -- <basename>`.
   - Clean (`  ` or `M `) ‚Üí in-place. Untracked (`??`) or dirty ‚Üí fallback: target = `<file>.compressed`.
   - `REJECTED` only on conflict/deleted states.
3. Resolve tier: `--tier` or ultra.
4. Apply tier rules for the selected tier inline (ultra: maximum reduction; full: balanced; lite: light trim).
5. Verify (file mode only, not blurb): compare compressed result against original. Every fact, rule, constraint must survive.
   - Recoverable (single dropped item or collapsed qualifier) ‚Üí fix in-place, report fix + suggest prevention.
   - Not recoverable (multi-element loss, structural damage, meaning reversal) ‚Üí restore original, `REJECTED: content lost ‚Äî <details>`.
5a. If target is `.md`: apply markdown lint checks inline to compressed result. Fix common issues in-place (trailing spaces, missing blank lines around fenced blocks, etc.).
   - All fixed ‚Üí record fixes. Some remain ‚Üí record fixes; list remaining as warnings. Nothing to fix ‚Üí record `hygiene: clean`.
   Hygiene issues never cause rejection ‚Äî caller is informed so no further pass needed.
6. Write result to target (in-place, `.compressed`, or `--target` path).

Output (required): `<before>‚Üí<after> bytes | <N>% reduction | <tier> | <mode>`
Mode = `in-place`, `alongside (<file>.compressed)`, or `source‚Üítarget (<dst>)`
If fixed: append `Fixed: <what>` + `Suggest: <prevention>`
If `.md`: append `Hygiene: clean` or `Hygiene: fixed <N> issue(s)` or `Hygiene: <N> warning(s) ‚Äî <details>`
If `--hygiene-verdict` provided: append `Source hygiene: <verdict-summary>` (one line summarising Phase 0 result)
Blurb mode: compressed text, then `---`, then reduction line.
