Params: `root` (required) — absolute path to dir with starting `skill.index`; `need` (required) — stated need as plain phrase (e.g. `compress text`, `build index`).

Procedure:
Follow in order. Stop at first stop condition.

1. Open `skill.index` at `root`. Missing → `no index here`, stop. Unreadable → `unreadable node` + path, stop.
2. Parse each line: `key: keyword, keyword, keyword`. Split at first colon — before = key, after (trimmed) = keywords split on `, `. Malformed (no colon, empty key, forbidden chars) → record in `inconsistencies`, skip. Other entries remain usable.
3. Check stamp: open `skill.index.sha256` at current node's dir. Compute SHA-256 of stored `skill.index` bytes. Match → overlay trusted. Mismatch or missing → overlay untrusted; record in `inconsistencies`. Raw index still usable; overlay disambiguation suppressed.
4. Match (case-fold): `need` appears as substring of entry key or any keyword → Hit. Collect all Hits.
5. Rank: self entry (`.`) or plain leaf (no trailing `/`) beats descent-marked (trailing `/`). Overlay trusted + two or more Hits tied → consult `skill.index.md` to break tie. Overlay may only disambiguate existing Hits, never produce new ones.
6. Evaluate:
   - One Hit → step 7.
   - No Hits → `no match`, stop.
   - Two or more tied after overlay → `ambiguous`, stop.
7. Resolve Hit:
   - Plain leaf (no trailing `/`): skill at `<current dir>/<key>/`. Return `hit` + path, stop.
   - Self entry (`.'): skill at current dir. Return `hit` + path, stop.
   - Descent-marked (trailing `/`): deeper index. Proceed to step 8.
8. Descend:
   Target path: single-segment `foo/` → `<current dir>/foo/`. Multi-segment / shortcut: walk each segment from current dir; at every step verify no `..`, no absolute-path indicator, no escape above invocation root. Escape → record `subtree-escape`, return `subtree-escape`, stop.
   Loop check: resolved target already in visited set → `reference loop` + ordered visited sequence, stop.
   Append to visited set. Open `skill.index` inside target. Missing → record `broken descent`, treat entry as absent, re-evaluate remaining from step 6. Exists → new current node, repeat from step 2.

Combo entry (descent-marked + target has self entry in its own `skill.index`):
1. Try as plain leaf first.
2. Leaf Hit → return it.
3. No leaf Hit → descend into sub-node, re-apply from step 2.

Entry types:
| Key form | Meaning |
| --- | --- |
| `foo` | Leaf skill at `<current dir>/foo/` |
| `foo/` | Sub-node: descend into `<current dir>/foo/skill.index` |
| `.` | Self entry: current dir is skill |
| `a/b/c/` | Shortcut with descent: walk path, open `skill.index` at `c/` |
| `a/b/c` | Shortcut leaf: skill at `c/`, no deeper index |

Stop conditions:
| Outcome | Condition |
| --- | --- |
| `hit` | One Hit survives; resolved path returned |
| `no index here` | `skill.index` absent at working dir |
| `no match` | No Hits at current node |
| `ambiguous` | Two or more Hits tied after overlay |
| `reference loop` | Descent lands on already-visited node |
| `subtree-escape` | Shortcut exits invocation root's subtree |
| `unreadable node` | `skill.index` exists but unreadable |
| `broken descent` | Descent target missing `skill.index`; entry treated as absent |

Visited-node tracking:
Maintain ordered list of every node visited on current resolution path. Node added when `skill.index` first opened. Track across all descent kinds: direct-child, shortcut endpoint, combo sub-node. Before opening any new node, check if dir already in list → reference loop. Scoped to current path; doesn't persist across crawls.

Overlay trust rules:
- Consult overlay (`skill.index.md`) only to break ties among existing Hits after raw-index ranking.
- Trusted only when stamp matches SHA-256 of raw `skill.index` bytes.
- Missing stamp or mismatch → untrusted; raw index only.
- Never produces new candidates.

Error handling:
- `skill.index` missing at root: `no index here`, stop.
- `skill.index` unreadable: `unreadable node` + path, stop.
- Malformed entry: record, skip, continue.
- Missing stamp: overlay untrusted; record, continue.
- Stamp mismatch: same as missing stamp.
- Broken descent: record, treat as absent, re-evaluate remaining.
- Shortcut intermediate segment missing: record `broken shortcut path` (first missing segment), treat as absent, continue.

Crawl report format:
```
## Crawl Report

need: <phrase>
root: <path>
outcome: hit | no match | ambiguous | reference loop | subtree-escape | no index here | unreadable node

### Hit
key: <entry key>
resolved path: <absolute path>
(omit section when outcome is not hit)

### Visited Nodes
1. <absolute path>
...

### Inconsistencies
- <type>: <description>
(omit if none)
```

Fail-fast rules:
- `root` missing or dir doesn't exist: `no index here`, stop.
- `need` empty: report missing parameter, stop.
- Never open files inside skill's folder (only `skill.index`, `skill.index.md`, `skill.index.sha256` permitted).
- Never climb above invocation root.
- Never follow descent marker not in current raw index.

Don'ts:
- Don't produce, author, or update any file.
- Don't open skill contents (only three index artifacts permitted).
- Don't climb above working dir.
- Don't trust overlay when stamp absent or mismatched.
- Don't follow shortcut out of invocation root's subtree.
- Don't consult network.
- Don't cache results across separate crawls.
