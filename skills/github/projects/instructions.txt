GH CLI Projects — GitHub Projects v2 via `gh project`: create, manage items, edit fields, archive, delete.

Projects use numeric IDs — resolve name → number before other commands:
```bash
gh project list --owner owner --format json --jq '.projects[] | select(.title=="My Project") | .number'
```

Create:
```bash
gh project create --owner owner --title "My Project"
gh project create --owner orgname --title "Project"
```

Add item (issue or PR) by URL:
```bash
gh project item-add PROJECT_NUM --owner owner --url https://github.com/owner/repo/issues/123
```

List items:
```bash
gh project item-list PROJECT_NUM --owner owner
```

Create field:
```bash
gh project field-create PROJECT_NUM --owner owner --name "Status" --data-type "SINGLE_SELECT"
```
Supported types: `TEXT`, `NUMBER`, `DATE`, `SINGLE_SELECT`, `ITERATION`.

Edit item field — single-select requires option ID, not label text; retrieve first:
```bash
gh project field-list PROJECT_NUM --owner owner --format json --jq '.fields[] | select(.name=="Status") | .options'
gh project item-edit PROJECT_NUM --owner owner --id ITEM_ID --field-id FIELD_ID --single-select-option-id OPTION_ID
```

Archive vs delete:
```bash
gh project item-archive PROJECT_NUM --owner owner --id ITEM_ID   # hides, keeps in project
gh project item-delete PROJECT_NUM --owner owner --id ITEM_ID    # permanent
```

Copy project as template:
```bash
gh project copy PROJECT_NUM --source-owner owner --target-owner owner --title "New Project"
```

Delete project:
```bash
gh project delete PROJECT_NUM --owner owner
```

Scope: `gh project` only. Excludes Projects v1 (classic boards), automation rules, and direct issue→project linking — use item-add for linking.
