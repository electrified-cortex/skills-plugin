Inputs: OWNER, REPO, PR_NUMBER, FILE_PATH, LINE_NUMBER, BODY, SIDE (default RIGHT — apply before steps 4–5), START_LINE (optional — multi-line only)

1. Auth: gh auth status 2>&1
2. Fetch SHA: gh pr view {PR_NUMBER} --repo {OWNER}/{REPO} --json headRefOid --jq '.headRefOid' → COMMIT_SHA
3. Verify file: gh pr diff {PR_NUMBER} --repo {OWNER}/{REPO} --name-only → FILE_PATH must appear; else stop.
4. Verify line (use the bundled tool — do not parse the diff manually):
   bash: bash verify-line-in-diff.sh {OWNER} {REPO} {PR_NUMBER} {FILE_PATH} {LINE_NUMBER} {SIDE}
   pwsh: pwsh verify-line-in-diff.ps1 {OWNER} {REPO} {PR_NUMBER} {FILE_PATH} {LINE_NUMBER} {SIDE}
   Exit 0 (IN_DIFF) → proceed.
   Exit 1 (NOT_IN_DIFF ranges:...) → stop; surface the listed valid ranges to the caller.
   Exit 2 (FILE_NOT_IN_DIFF) → stop; file has no changes in this PR.
   Exit 3 (USAGE_ERROR) → check invocation. Exit 4 (API_ERROR) → surface error to caller.
   WINDOWS: Never use a leading `/` in `gh api` paths — Git Bash rewrites `/repos/...` as a filesystem path. Use `repos/...` not `/repos/...`.
5. Dedup: gh api --paginate repos/{OWNER}/{REPO}/pulls/{PR_NUMBER}/comments --jq '.[] | select(.path=="{FILE_PATH}" and .line=={LINE_NUMBER} and .side=="{SIDE}") | {id,body}' → match → return {status:"duplicate", comment_id:<id>, comment_url:"https://github.com/{OWNER}/{REPO}/pull/{PR_NUMBER}#discussion_r<id>"}
6. Post (single): gh api --method POST repos/{OWNER}/{REPO}/pulls/{PR_NUMBER}/comments --field body="{BODY}" --field commit_id="{COMMIT_SHA}" --field path="{FILE_PATH}" --field line={LINE_NUMBER} --field side="{SIDE}"
   Post (multi, START_LINE set): add --field start_line={START_LINE} --field start_side="RIGHT"; use side="RIGHT". Never use `position`.
7. Return: { "status": "posted"|"duplicate"|"error", "comment_id": <id or null>, "comment_url": "<https://github.com/{OWNER}/{REPO}/pull/{PR_NUMBER}#discussion_r<id> or null>", "message": "<one line>" }
   On 422 field=pull_request_review_thread.line message="could not be resolved": line is not in diff — report line not commentable.
   On other 422: surface full error body to caller.
   On any other error: { "status": "error", "comment_id": null, "comment_url": null, "message": "<error description>" }
